# OpenMP
- ფაილი შეიქმნა

  ```%%writefile hello_openmp.cpp```
- კომპილაცია
  
  ``` !g++ hello_openmp.cpp```

- გაშვება
 
    ```!./hello_openmp```

-  	#include <omp.h>
  ``` აერთებს OpenMP სათაურ ფაილს. მის გარეშე ვერ გამოვიყენებთ omp_get_thread_num(), omp_get_num_threads() და სხვა OpenMP ფუნქციებს. ```
- #pragma omp parallel

  ```OpenMP დირექტივაა. კომპილატორს ეუბნება, რომ მომდევნო ბლოკი უნდა შესრულდეს პარალელურად რამდენიმე ნაკადში.```
- { 
  ```იწყება პარალელური რეგიონი. ამ რეგიონში შემავალი კოდი ერთდროულად გაეშვება რამდენიმე ნაკადის მიერ.```
- cout << "Hello from thread " << omp_get_thread_num()
  ``` ბეჭდავს ტექსტს და მიმდინარე ნაკადის ნომერს. omp_get_thread_num() აბრუნებს იმ ნაკადის იდენტიფიკატორს, რომელიც ამ ხაზს ასრულებს ```
- << " of " << omp_get_num_threads() << endl;
  ```ამატებს მთლიან ნაკადების რაოდენობას. omp_get_num_threads() გვიჩვენებს რამდენი ნაკადი შეიქმნა მოცემულ პარალელურ რეგიონში. endl ხაზის ბოლოს გადასვლას და buffer-ის გასუფთავებას აკეთებს. ```
- }
  ```ასრულებს პარალელურ რეგიონს. აქ დასრულების შემდეგ დამატებითი ნაკადები ერთიანდებიან.```
- return 0;
  ```ოპერაციულ სისტემას ატყობინებს, რომ პროგრამა წარმატებით დასრულდა.```
- }
  ```ასრულებს main ფუნქციას.```

- omp_set_num_threads(4);
    ```OpenMP-ს მიუთითებს, რომ შემდეგ პარალელურ რეგიონში სასურველია 4 ნაკადის გამოყენება. ეს არის მოთხოვნა და გარემომ ზოგჯერ შეიძლება სხვა რაოდენობა აირჩიოს, მაგრამ ჩვეულებრივ მიიღება 4. ```
- cout << "Thread ID: " << omp_get_thread_num()
  ```ბეჭდავს კონკრეტული ნაკადის ნომერს.```

- << ", Total threads: " << omp_get_num_threads() << endl;
  ``` ამავე სტრიქონზე აჩვენებს მთლიან ნაკადების რაოდენობას, რაც სტუდენტს ეხმარება გადაამოწმოს, რამდენი ნაკადი შეიქმნა რეალურად.```
# 3 parallel for და იტერაციების განაწილება
-  	for (int i = 0; i < 12; i++) {
  ``` ციკლი 12 იტერაციით. OpenMP ამ იტერაციებს ნაწილებად ყოფს და ავალებს სხვადასხვა ნაკადს.```
- cout << "Iteration " << i
  ``` ბეჭდავს, რომელი იტერაცია სრულდება ამჟამად.```
  - << " executed by thread " << omp_get_thread_num() << endl;
  ``` ამატებს იმ ნაკადის ნომერს, რომელმაც ეს კონკრეტული იტერაცია შეასრულა.```
# Race Condition და atomic გამოსწორება
- for (int i = 0; i < 100000; i++) {
  ``` ციკლი 100000-ჯერ იმუშავებს, რათა კონკურენციის ეფექტი აშკარა გახდეს. ```
  -  	#pragma omp atomic
  ``` ეუბნება OpenMP-ს, რომ მომდევნო მარტივი მეხსიერებითი ოპერაცია უნდა შესრულდეს ატომურად, ანუ უსაფრთხოდ, ისე რომ სხვა ნაკადმა შუაში არ ჩაერიოს.```
-  	counter++;
  ```ზრდის საერთო მთვლელს ერთით. atomic-ის გარეშე ეს ხაზი პრობლემურია, რადგან რამდენიმე ნაკადს შეუძლია ერთდროულად წაიკითხოს, შეცვალოს და ჩაწეროს counter.```
  -  	cout << "Counter = " << counter << endl;
  ```ბეჭდავს საბოლოო შედეგს. atomic-ის გამოყენებისას მოსალოდნელია ზუსტად 100000. ```
  
#  Reduction-ით ჯამის გამოთვლა
-  #pragma omp parallel for reduction(+:sum)
  ```აქ ხდება ორი მნიშვნელოვანი რამ: for ციკლი ნაწილდება ნაკადებზე და sum ცვლადისთვის ინიშნება reduction რეჟიმი. თითოეულ ნაკადს ექნება თავისი დროებითი sum, ხოლო ბოლოს OpenMP ამ შედეგებს შეაჯამებს + ოპერატორით.```
